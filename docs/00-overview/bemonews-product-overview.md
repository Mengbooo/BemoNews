# bemoNews Product Overview

## 1. 产品定位

bemoNews 是一个面向个人使用的信息获取软件，当前阶段的核心目标不是做一个大而全的信息平台，而是做一个高频可用、低噪音、可持续迭代的个人情报台。

它服务于“开发相关信息获取”这一主要场景，帮助使用者更高效地收集、过滤、整理和消费外部信息，并以更轻量的方式形成稳定的信息输入节奏。

在产品形态上，bemoNews 优先是一个资讯产品，而不是决策辅助工具。MVP 阶段重点解决“我今天该看什么”这个问题，而不是“我接下来该做什么”。

## 2. 产品目标

bemoNews 希望实现一套对个人友好的信息流处理机制：

- 自动收集多个信息源的内容
- 对内容进行初步加工，尽可能减少噪音、标题党和低价值重复信息
- 在前端以适合快速浏览和深入阅读的方式展示
- 通过定时邮件主动推送每日信息报告
- 支持使用命令即时生成最新报告

它的价值不在于“收集得越多越好”，而在于在尽量不遗漏重要信息的前提下，帮助使用者节省筛选时间。

## 3. 核心体验

bemoNews 的理想使用体验如下：

- 每天固定时间，系统自动生成并推送一份邮件简报
- 使用者也可以在任意时间通过一个命令主动生成最新报告
- 在摸鱼、碎片时间里，可以快速浏览速报版本
- 在需要深入了解时，可以查看更完整的详细报告版本

这意味着产品天然需要支持两种阅读形态：

- 速报：强调快速扫读，适合碎片时间消费
- 详报：强调信息完整度，适合集中阅读

晚间回顾不作为当前阶段的重点能力。

## 4. 用户与场景

bemoNews 当前服务于单个个人用户，主要使用场景是开发工作相关的信息获取，尤其适合在工作间隙、摸鱼时间或日常信息巡检时使用。

首期关注的内容方向以开发相关为主，但产品应从一开始就预留“可配置的信息主题”能力，为后续扩展到更多信息对象做好准备。

未来可覆盖的信息对象包括但不限于：

- 开发与技术动态
- AI 资讯与模型更新
- 行业趋势
- 产品与创业动态
- 投资机会相关信息
- 特定公司、人物或主题追踪

## 5. 产品形态

bemoNews 的 MVP 形态是一套“个人情报台”：

- 有主动推送能力
- 有报告生成能力
- 有前端页面承载浏览体验
- 有信息加工环节作为核心竞争力

在更长远的形态上，bemoNews 可以演进为带有情报助理属性的产品，例如在前端中提供一个可对报告内容进行追问的小窗口，并接入用户自己的 AI API。

但这一能力不纳入 MVP。现阶段用户完全可以通过复制链接或内容到外部 AI 工具中继续提问，因此没有必要在第一阶段承担较重的交互与模型集成复杂度。

## 6. 信息处理流程

bemoNews 的核心流程应为：

`收集信息 -> 加工信息 -> 前端展示 -> 主动推送`

其中，“加工信息”是整个产品最重要的环节，也是后续体验差异化的基础。

加工环节的职责包括：

- 去除明显噪音
- 降低标题党内容的干扰
- 减少重复信息
- 保留较高价值的信息线索
- 为速报和详报提供不同粒度的内容组织

在 MVP 阶段，这一加工标准可以相对放宽。优先目标不是强力压缩信息，而是尽量保留更多有效内容，以便调试和观察什么样的筛选策略最适合后续迭代。

换句话说，MVP 阶段允许“有一点杂”，但不能“失真”或“过度自作主张”。

## 7. 信息源策略

bemoNews 的首期信息源以 RSS 为主。

这样做有几个明显好处：

- 接入成本低
- 更新机制成熟
- 便于统一采集和调试
- 有利于快速完成 MVP

在后续阶段，bemoNews 可以扩展到无法直接通过 RSS 订阅的信息源，例如：

- Twitter / X
- 小红书
- 其他平台型内容源

这部分未来可以考虑借助 agent 或相关 MCP 能力进行补充采集，但不纳入当前 MVP 范围。

## 8. 内容风格与产品个性

bemoNews 的内容风格应保持：

- 冷静
- 中立
- 克制
- 反噪音

它不追求情绪化表达，不放大标题，不制造紧迫感，也不刻意加入主观煽动性的判断。

它更像一个认真做信息整理的人，帮助用户把值得看的内容留下来，把低质量信息尽量挡在外面。

## 9. MVP 范围

bemoNews 的 MVP 重点是验证“个人情报台”是否成立，范围应控制在以下几类能力内：

- 基于 RSS 的信息采集
- 基础的信息过滤与整理
- 两种报告形态：速报与详报
- 前端页面展示报告内容
- 基于定时任务的邮件推送
- 通过命令手动生成最新报告

MVP 明确不包含以下内容：

- 内置 AI 对话助理
- 面向决策的深度建议能力
- 复杂的知识沉淀与长期记忆系统
- 非 RSS 平台的大规模采集接入
- 机会发现、投资判断等高阶分析能力

## 10. 后续演进方向

在 MVP 验证完成后，bemoNews 可以继续向以下方向演进：

- 更精细的信息加工与去重策略
- 可配置的信息主题与权重体系
- 对非 RSS 信息源的扩展采集
- 基于报告内容的 AI 追问入口
- 从“资讯获取”逐步扩展到“机会发现”和“知识沉淀”

整体上，bemoNews 的演进路径应当是：

`个人情报台 -> 更智能的信息加工系统 -> 轻量情报助理 -> 更长期的认知支持工具`

## 11. 一句话总结

bemoNews 是一个面向个人使用的、以开发相关信息为起点的低噪音情报台，通过 RSS 收集内容、对信息进行初步加工，并以“前端可读 + 邮件推送 + 命令生成”的方式，帮助用户建立稳定、轻量、可持续的信息获取节奏。
